# Git操作の確認

`git add`・`git commit`・`git push` を実行する前に、必ず以下の確認を行う：

1. 「〇〇します。よろしいですか？」と内容を明示して聞く
2. ユーザーから「はい」の返答を得てから実行する

**いかなる場合も、確認なしにgit操作を実行しないこと。**

## git push の確認は特に厳守

`git push` はリモートリポジトリに影響するため、commit と push をまとめて確認する場合も含め、
**必ずユーザーの「はい」を得てから push を実行すること。**
「コミットしてよいですか？」への「はい」は push の承認ではない。push は別途確認すること。
